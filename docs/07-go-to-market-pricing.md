# Go-To-Market and Pricing

## Target Market

### Beachhead

- AI-native software startups
- platform engineering teams
- teams already spending materially on AI coding
- SMB software companies that want AI leverage without enterprise-scale token waste

### Expansion

- mid-market engineering orgs
- enterprises with internal AI engineering tooling
- regulated teams needing safer AI delivery controls

## Positioning Statement

For software teams that use AI to write code, `be-ai-heart` is the context operating layer that gives agents durable project memory, reduces token waste, and keeps code changes aligned with architecture.

## Wedge Strategy

Sell on cost and quality together.

If you only sell “better context,” the value is abstract.
If you only sell “AI safety,” the product feels defensive.

The stronger narrative:

- reduce token spend
- reduce duplicate work
- reduce review cleanup
- increase trust in AI-generated changes
- preserve project intent from code and documents

## Pricing Model

### Free

- local CLI
- single-user local indexing
- basic MCP
- limited benchmark scenarios

Use:

- developer adoption
- bottom-up distribution

### Team

- per-seat subscription
- shared workspace
- benchmark reports
- policy packs
- usage analytics

Indicative starting point:

- `49-99 USD` per active developer per month

For SMBs, pricing should feel obviously cheaper than the monthly waste from:

- avoidable token burn
- review cleanup
- duplicate implementation
- rework caused by missing requirements or architecture context

### Enterprise

- annual license
- SSO/SAML
- audit logs
- admin control plane
- VPC/on-prem
- dedicated support
- custom benchmark engagement

Indicative range:

- `25k-150k+ USD` annually depending on seats, repos, and deployment model

## Packaging

Primary packaging dimensions:

- active developers
- indexed repositories
- benchmark/reporting features
- deployment model

## Acquisition Channels

- founder-led outbound to AI-heavy engineering leaders
- benchmark-led demos
- technical content and case studies
- GitHub and developer community distribution via free CLI
- partner integrations with AI tool ecosystems

## Early GTM Assets To Build

- product website
- short demo video
- benchmark whitepaper
- docs portal
- pilot offer deck
- security overview

## Pilot Offer

Recommended offer:

- 2-week design partner engagement
- 1-3 repositories
- before/after benchmark
- executive report at the end

Pilot success criteria:

- clear token delta
- clear reduction in duplicate work
- at least one engineering leader champion

## Sales Funnel

1. Developer finds CLI or content
2. Team tries local benchmark
3. Demo call for shared/team use
4. Pilot
5. Annual contract

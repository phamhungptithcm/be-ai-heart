# Protect the heart

Security is not a later enterprise add-on. It has to shape how project memory is captured, stored, and exposed from the start.

## Minimize what enters context

Do not push sensitive paths, secrets, or private payloads into the heart unless they are explicitly needed and governed.

## Use policy to constrain reuse

The policy layer exists to keep AI from crossing module boundaries, reviving deprecated paths, or inventing new implementations when a preferred reuse path already exists.

## Separate surfaces clearly

Public website, customer portal, and internal admin should never blur together. Buyers need trust; operators need auditability.

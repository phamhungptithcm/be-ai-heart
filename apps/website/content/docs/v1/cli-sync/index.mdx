# Run the local memory loop

The CLI is the fastest way to build trust because every important step is explicit.

## Scan source and documents

Use `heart scan` after meaningful repository changes so the persisted heart stays aligned with code, documents, and linked context.

## Generate diagrams for review

You can produce symbol graph, high-level, class, and sequence-style diagrams. The point is not decoration; it is faster repo legibility for engineers, reviewers, and buyers.

## Sync into the portal and admin

Publishing should happen only after the local artifacts look credible. The portal is for the customer workspace; the admin surface is for owner operations and support.

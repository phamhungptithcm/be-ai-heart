# Benchmark before rollout

The strongest sales motion for `be-ai-heart` is not “our graph looks smart”. It is “this repository now needs less AI spend for the same or better outcome”.

## Use the same repo and task

Keep the repo snapshot, task statement, and model class fixed between baseline and heart-assisted runs.

## Report both savings and quality

Track token savings, direct cost savings, memory refresh reduction, review edit reduction, and architecture quality together. Lower spend is not enough if the output gets worse.

## Publish the report

After the comparison, publish the benchmark so the portal shows the customer-facing ROI view and admin shows the owner-facing revenue and support view.

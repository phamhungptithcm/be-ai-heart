# What BeHeart is

BeHeart is durable project memory for AI-assisted software teams. It helps coding agents understand the repository, project documents, architecture rules, and benchmark evidence before they generate code.

The product is not an IDE, a model wrapper, or a generic vector search tool. It is a context operating layer that feeds existing AI coding tools with better memory.

## The problem it solves

AI coding tools often start cold. They miss existing modules, ignore requirements, repeat discovery work, cross architecture boundaries, and make teams pay tokens to re-explain the same project context.

BeHeart makes that context durable so the next AI task starts closer to the real system.

## Core services

- Code Graph maps symbols, files, modules, dependencies, tests, and likely impact paths.
- Document Memory links PRDs, architecture docs, user stories, decisions, and business requirements.
- Policy Rails expose preferred reuse paths, deprecated areas, boundaries, and sensitive exclusions.
- CLI + MCP Runtime delivers memory locally to developer workflows and AI tools.
- Portal + Admin Surfaces separate customer workspace from internal operations.
- Benchmark ROI measures whether BeHeart reduced cost, duplicate work, review cleanup, and risk.

## Who uses it

- Engineers use the CLI and MCP tools before asking AI to code.
- Tech leads use graph, docs, policy, and benchmark evidence to reduce review cleanup.
- Managers use ROI reports to decide whether AI rollout should expand.
- Internal operators use admin views for support, revenue, retention, and tenant risk.

## What success looks like

A successful BeHeart rollout shows current repo memory, cited context packs, visible docs freshness, policy warnings, generated diagrams, and benchmark evidence that is clearly labeled as observed, estimated, or mixed.

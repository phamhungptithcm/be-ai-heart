# GitHub Workflow

## Objective

Keep repository collaboration clear, reviewable, and lightweight enough for a small startup team.

## Workflow

1. Prefer small, reviewable changesets.
2. Make PR descriptions explicit about scope, risk, and validation.
3. Keep issue templates aligned with benchmark, product, and engineering needs.
4. Preserve a clean path from design partner feedback to shipped change.

## Guardrails

- do not create ceremony that slows early product learning
- keep labels and templates minimal but useful
- release notes should emphasize value, risk, and migration impact

## Deliverables

- repository workflow changes
- templates or automation updates when needed
- clear review expectations

# Project Owner

## Read First

- `docs/00-executive-summary.md`
- `docs/02-prd.md`
- `docs/08-roadmap-operating-model.md`

## Objective

Maximize proof of value with the least delivery risk.

## Workflow

1. Frame the user outcome and business outcome.
2. Choose the smallest shippable slice that advances trust or ROI proof.
3. Write explicit acceptance criteria.
4. Call out dependencies, risks, and what is intentionally deferred.

## Prioritization Order

1. trust and correctness
2. measurable savings
3. adoption ease
4. enterprise controls
5. expansion features

## Deliverables

- scope statement
- acceptance criteria
- sequencing rationale
- risk and dependency notes

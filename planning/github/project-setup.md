# Project Setup

## GitHub Project Fields

- `Status`: Todo, In Progress, In Review, Done, Blocked
- `Priority`: P0, P1, P2
- `Milestone`: M0 Foundation, M1 Heart Core MVP, M2 Agent Runtime and Benchmark, M3 Product Surfaces, M4 Enterprise Readiness
- `Track`: Heart Core, Runtime, Product Surface, Enterprise
- `Risk`: Low, Medium, High

## Views

- Roadmap grouped by milestone
- Board grouped by status
- Table grouped by track
- Security table filtered by `type:security`

## Initial Workflow Rule

- Every issue must have:
  - one priority
  - one milestone
  - one track
  - at least one type label

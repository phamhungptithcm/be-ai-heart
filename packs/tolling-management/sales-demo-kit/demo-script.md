# Demo Script

## 5-Minute Executive Demo

Audience: CTO, operations leader, founder, investor.

1. Open with the problem: tolling buyers need to see operational depth before funding implementation.
2. Show the microsite hero and module overview.
3. Open Agent Account 360 and explain the call-center storyline.
4. Show the roadside-to-back-office workflow.
5. Show security/payment posture and ROI hypothesis.
6. Close with next step: customize the overlay for the buyer's agency and run a 30-minute workflow review.

## 15-Minute Product Demo

Audience: operations, customer service, product owner.

1. Search for `DEMO-ACCT-1001`.
2. Verify the customer in the Account 360 preview.
3. Explain `TRIP-DEMO-7002` from plate review through invoice line.
4. Show funds/payment panel with tokenized payment placeholder.
5. Open `CASE-DEMO-301` dispute flow and show SLA/evidence placeholder.
6. Show `CASE-DEMO-302` transponder replacement and fulfillment state.
7. Show notification center with scam-safe guidance.
8. Show reports dashboard and implementation roadmap.

## 30-Minute Technical Demo

Audience: CTO, architect, implementation partner.

1. Explain pack inputs: core domain, regional layer, agency overlay, customer overlay.
2. Show architecture diagram and service map.
3. Show DB draft and ERD.
4. Walk event flow: raw event, normalized event, trip, rating, posting, invoice/violation, case, payment, report.
5. Discuss security boundaries: PII, plate image, hosted payment, audit, tenant overlay.
6. Show source claims and unsupported-claim rules.
7. Show benchmark plan and measurement path.
8. End with discovery questions for agency policy, integrations, migration, and rollout.

## Customer Service Call Scenario

Trigger: Customer asks why they received a bill.

1. Agent searches by `DEMO-ACCT-1001` or plate `DEMO123`.
2. Agent verifies identity.
3. Agent reviews invoice `INV-DEMO-9001`.
4. Agent explains trip source, tag/plate match, and invoice line.
5. Agent offers safe actions: pay bill, open dispute, update vehicle/tag info, send official payment guidance.
6. Agent records a case note and audit reason.

## Dispute Handling Scenario

Trigger: Customer disputes managed lane charge `TRIP-DEMO-7002`.

1. Open case `CASE-DEMO-301`.
2. Review event metadata and fake evidence placeholder.
3. Confirm policy is agency-specific.
4. Choose action: request evidence, escalate, resolve, or apply approved adjustment.
5. Record decision, reason, actor, and notification state.

## Transponder Replacement Scenario

Trigger: Tag `TAG-DEMO-0007` is not reading.

1. Agent sees plate-read pattern in Account 360.
2. Agent opens transponder issue case.
3. Fulfillment reserves `TAG-DEMO-0102`.
4. System creates replacement order and shipment placeholder.
5. Old tag state changes to replacement pending.
6. Agent sends notification without unsafe payment links.

## Payment/Funds Scenario

Trigger: Auto-replenishment failed for `DEMO-ACCT-1002`.

1. Agent verifies identity.
2. Payment panel shows `PAY-DEMO-FAIL-01`.
3. Agent explains account state and safe payment options.
4. Customer uses hosted payment placeholder.
5. Ledger entry and receipt placeholder are shown.
6. Any refund or adjustment above threshold moves to approval-needed state.
